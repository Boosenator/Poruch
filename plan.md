# Поруч: продуктовий план

## 1. Позиціонування

**Поруч** — український навігатор по перевірених місцях, сервісах і спільнотах у Чехії.

Продукт має відповідати на просте питання: “Куди мені звернутися поруч?” Не просто показати карту, а допомогти людині швидко знайти зрозуміле, безпечне й актуальне місце: лікаря, консультацію, курси, роботу, кафе, магазин, подію або спільноту.

### Для кого

- Українці, які щойно приїхали до Чехії.
- Українці, які вже живуть у Чехії, але шукають перевірені сервіси.
- Батьки, студенти, працівники, підприємці, волонтери.
- Організації й бізнеси, які працюють з українською аудиторією.

### Обіцянка продукту

“Поруч” економить час, зменшує тривогу й дає практичну відповідь українською мовою.

## 2. Поточний стан

Є базовий Next.js 14 продукт з мок-даними:

- Головна карта на Mapbox.
- Список місць і фільтри категорій.
- Деталі місця на десктопі й мобільному.
- Мобільний UX з bottom sheet.
- Маршрути: вибір між внутрішнім маршрутом на карті та зовнішніми апками.
- Сторінки `/list`, `/saved`, `/suggest`, `/place/[id]`.
- Supabase клієнти підготовлені, але реальна БД ще не підключена.
- Мок-дані вже наближені до реальності: офіційні/соціальні ресурси + gastro-точки з `uagastromapa.cz`.

## 3. Продуктові принципи

1. **Спочатку користь, потім краса.**
   Користувач має знайти місце за 10-20 секунд.

2. **Перевіреність важливіша за кількість.**
   Краще 80 актуальних місць, ніж 800 випадкових.

3. **Мобільний сценарій головний.**
   Людина часто шукає “тут і зараз” з телефону.

4. **Українська як основна мова.**
   Чехія присутня через адреси, категорії, офіційні назви та майбутню чеську локалізацію.

5. **Карта не є вся суть продукту.**
   Потрібні списки, SEO-сторінки місць, добірки, форми додавання, модерація, профілі, збережені.

## 4. Основні сценарії

### Сценарій 1: “Мені треба знайти допомогу”

Користувач відкриває сайт, бачить карту, вибирає категорію, натискає місце, читає короткі деталі, телефонує або будує маршрут.

Критично:

- швидкий пошук;
- зрозумілі категорії;
- видима верифікація;
- телефон, сайт, маршрут в один клік.

### Сценарій 2: “Я хочу зберегти місця”

Користувач додає місце в збережені й повертається пізніше.

MVP може бути localStorage. Після auth — Supabase saves.

### Сценарій 3: “Я знаю корисне місце”

Користувач або власник бізнесу додає місце через форму. Місце йде на модерацію.

Критично:

- проста форма;
- пояснення, що місце з’явиться після перевірки;
- контакт для уточнення.

### Сценарій 4: “Мені потрібна добірка”

Наприклад:

- “Де в Празі говорять українською”
- “Українські кафе”
- “Безкоштовна правова допомога”
- “Курси чеської”
- “Для батьків з дітьми”

Це важливо для SEO, соцмереж і реальної навігації.

## 5. MVP 1: публічний корисний сайт

Ціль: сайт уже можна показувати людям і збирати фідбек.

### Функції

- Карта з реальними мок/seed-даними.
- Список місць.
- Пошук за назвою, адресою, тегами.
- Фільтр категорій.
- Детальна сторінка місця.
- Маршрут: на сайті або в Google Maps / Apple Maps / Mapy.cz.
- Збережені місця в localStorage.
- Форма “Додати місце”.
- Базова SEO-структура для сторінок місць.
- Сторінка “Про Поруч”.
- Сторінка “Контакти / Партнерство”.

### Не входить у MVP 1

- Реальні акаунти користувачів.
- Публічні відгуки.
- Повна адмінка.
- Автоматичний імпорт даних.
- Платні функції.

## 6. MVP 2: реальна база й модерація

Ціль: перейти з моків на Supabase і мати керований каталог.

### Supabase

Таблиці:

- `places`
- `suggestions`
- `profiles`
- `saves`
- `reviews` пізніше

### Дані

Поля для `places`:

- назва;
- категорія;
- підкатегорія;
- місто;
- адреса;
- координати;
- телефон;
- сайт;
- Instagram/Facebook;
- email;
- опис;
- мови;
- години;
- теги;
- фото;
- verified/active/free;
- source/source_url;
- дата останньої перевірки.

### Модерація

Статуси:

- `pending`
- `needs_info`
- `approved`
- `rejected`
- `archived`

Адмін має бачити:

- нові пропозиції;
- зміни до існуючих місць;
- місця без координат;
- місця, які давно не перевірялися.

## 7. Категорії

Поточні:

- Медицина
- Право
- Мова
- Робота
- Житло
- Магазини
- Кафе
- Спільнота

Рекомендовано додати підкатегорії:

- Медицина: сімейний лікар, педіатр, психолог, стоматолог, аптека.
- Право: документи, візи, соціальна допомога, переклади.
- Мова: курси, розмовні клуби, бібліотеки.
- Робота: вакансії, консультації, коворкінги.
- Житло: оренда, гуртожитки, кризове житло.
- Магазини: продукти, книги, сервіси, перекази.
- Кафе: ресторан, кафе, кондитерська, доставка.
- Спільнота: події, школи, діти, волонтерство.

## 8. UX план

### Мобільний

Пріоритет:

- карта на весь екран;
- фільтри компактно зверху;
- список через bottom sheet;
- деталі місця через bottom sheet;
- маршрут без втрати контексту;
- sticky actions у деталях.

Потрібно доробити:

- нормальне меню замість “бургер без функції”;
- перемикання “Карта / Список / Збережені” в мобільному меню;
- кнопка “Про Поруч” і “Додати місце” в мобільному меню;
- закриття route menu при кліку поза ним;
- стан “немає результатів” прямо на карті.

### Десктоп

Пріоритет:

- ліва панель: пошук, фільтри, список;
- центр: карта;
- права панель: деталі;
- окремі сторінки для SEO.

Потрібно доробити:

- краща щільність списку;
- сортування: поруч, популярні, перевірені, безкоштовні;
- фільтр “говорять українською”;
- фільтр “безкоштовно”;
- фільтр “відчинено зараз” після нормалізації годин.

## 9. Контент і дані

### Початкові джерела

- Офіційні сайти: посольство, міські установи, інтеграційні центри.
- NGO: Charita, InBáze, CIC, ICP.
- Українські бізнеси: GastroMapa, Google Maps, Instagram, сайти.
- Спільноти: Facebook/Telegram, але тільки після ручної перевірки.

### Політика якості

Кожне місце має мати мінімум:

- назву;
- категорію;
- місто;
- адресу або координати;
- короткий опис українською;
- джерело;
- дату останньої перевірки.

Для verified:

- перевірений сайт або соцмережа;
- контакт або офіційна сторінка;
- координати збігаються з адресою.

## 10. SEO і сторінки

Потрібні сторінки:

- `/` — карта як головний продукт.
- `/list` — каталог.
- `/place/[id]` — SEO-сторінка місця.
- `/suggest` — додати місце.
- `/saved` — збережені.
- `/about` — що таке Поруч.
- `/partners` — для бізнесів і організацій.
- `/collections/[slug]` — добірки.
- `/city/[city]` — майбутня сторінка міста.
- `/category/[category]` — SEO-категорії.

Перші добірки:

- Українські кафе в Празі.
- Безкоштовна допомога українцям у Празі.
- Курси чеської мови.
- Куди звернутися з документами.
- Місця для родин з дітьми.

## 11. Довіра і безпека

Потрібно явно показувати:

- verified badge;
- дату останньої перевірки;
- джерело інформації;
- кнопку “Повідомити про помилку”;
- дисклеймер для медичних/правових місць: інформація довідкова, перевіряйте деталі напряму.

Для відгуків:

- не запускати без модерації;
- спочатку простий “корисно / неактуально”;
- потім короткі відгуки з auth.

## 12. Аналітика

Події:

- search_performed
- category_selected
- place_opened
- route_clicked
- call_clicked
- website_clicked
- save_clicked
- suggest_started
- suggest_submitted
- report_issue_clicked

Метрики:

- кількість відкриттів місць;
- конверсія в дзвінок/маршрут/сайт;
- популярні категорії;
- пошукові запити без результатів;
- частка мобільних користувачів;
- кількість запропонованих місць.

## 13. Майбутня монетизація

Не ставити монетизацію попереду довіри.

Можливі моделі:

- партнерські профілі для бізнесів;
- пріоритетне оновлення інформації;
- промо добірки з явним маркуванням;
- гранти / підтримка NGO;
- B2B сторінки для українських підприємців;
- спонсорство корисних категорій.

Не робити:

- приховану рекламу;
- платну “верифікацію” без реальної перевірки;
- продаж персональних даних.

## 14. Технічний roadmap

### Етап A: стабілізація мок-продукту

- Перевірити всі мобільні сценарії.
- Доробити бургер-меню.
- Винести великі компоненти з `src/app/page.tsx`.
- Зробити localStorage для збережених.
- Поліпшити `/suggest`.
- Додати `/about` і `/partners`.
- Додати empty states і loading states.

### Етап B: Supabase

- Створити SQL migrations.
- Додати seed із поточного `MOCK_PLACES`.
- Зробити server queries для places.
- Підключити suggestions form до БД.
- Додати Supabase Storage для фото.
- Додати RLS.
- Додати admin-only модерацію.

### Етап C: Auth

- Supabase Auth email magic link.
- Google OAuth.
- Профіль користувача.
- Saves у БД.
- Role `admin`.

### Етап D: SEO і контент

- Metadata для кожної сторінки місця.
- JSON-LD для LocalBusiness / Organization.
- Sitemap.
- Категорійні сторінки.
- Добірки.
- Open Graph image.

### Етап E: операційна якість

- Панель “потрібно перевірити”.
- Поле `last_verified_at`.
- Звіти про помилки.
- Історія змін місця.
- Імпорт із CSV/JSON.

## 15. Найближчі завдання

1. Перевірити UX головної карти на мобільному після додавання 26 місць.
2. Зробити повноцінне мобільне меню.
3. Перенести збережені з state у localStorage.
4. Додати сторінку `/about`.
5. Додати сторінку `/partners`.
6. Переробити `/suggest` у реальну форму з полями майбутньої БД.
7. Додати кнопку “Повідомити про помилку” в деталях місця.
8. Винести `PlaceDetails`, `MobilePlaceDetails`, `RouteActions`, `ListCanvas` в окремі компоненти.
9. Підготувати `supabase/migrations/001_initial_schema.sql`.
10. Підготувати seed-скрипт із поточних мок-даних.

## 16. Ризики

- Дані швидко старіють.
- Координати й адреси можуть бути неточними.
- Медична й правова інформація чутлива.
- Без модерації відгуки можуть стати джерелом шуму.
- Якщо карта стане єдиним інтерфейсом, SEO і discovery постраждають.

## 17. Рішення на зараз

На найближчий цикл фокус:

1. Зробити сайт корисним навіть без акаунта.
2. Дати якісний мобільний досвід.
3. Підготувати реальну БД.
4. Почати збирати пропозиції місць.
5. Додати довіру: джерело, verified, report issue, last checked.
